面试官最爱的一道题：

```js
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
// 输出：3 3 3，而不是 0 1 2
```

## 为什么是 3 3 3

`var` 声明的 `i` 属于**函数作用域**，三个回调闭包捕获的是同一个变量。等到定时器触发时，循环早已结束，`i` 已经变成 3。

闭包记住的不是值，而是**变量本身的引用**——这是理解一切闭包问题的钥匙。

## 三种修法

1. 用 `let`：块级作用域让每轮循环都有一个全新的 `i`；
2. 用 IIFE 手动制造一层作用域：

```js
for (var i = 0; i < 3; i++) {
  ((j) => setTimeout(() => console.log(j), 100))(i);
}
```

3. 利用 `setTimeout` 的第三个参数把值传进去。

## 闭包的正经用途

| 场景 | 例子 |
| --- | --- |
| 私有状态 | 计数器、模块模式 |
| 柯里化 | `add(1)(2)(3)` |
| 防抖节流 | 记住上一次的定时器句柄 |
| 事件回调 | 记住绑定时的上下文 |

> 闭包不是奇技淫巧，而是「函数 + 它出生时的环境」这件再自然不过的事。
